# AI 内容运营中台架构文档（TurboPush + IMA，轻量版 SQLite）

## 一、架构定位

- **目标**：基于 TurboPush 二次开发，构建全流程智能内容运营中台。

- 核心价值

  ：

  - 内置 Agent 执行能力
  - 账号级内容风格和多平台版本管理
  - 审核流闭环
  - 数据复盘分析
  - 知识搜索直接调用 IMA，无需自研知识库

## 三、模块职责

### 1. 内容资产中心

- 管理选题、标题、正文、封面、标签、素材、发布记录
- 支持多平台版本映射
- 提供 API 给 Agent 调用和审核中心读取

### 2. Agent Runtime

- **Task Engine**：任务调度
- **Prompt Engine**：Prompt 模板管理
- **Tool Engine**：调用 IMA 搜索、素材库、图片生成、审核服务、平台规则
- **Model Gateway**：统一调用模型接口（OpenAI/Gemini/DeepSeek/Qwen 等）
- **去掉 Memory Engine**：不存历史生成记录

### 3. 审核中心

- 支持状态机：
  - 待生成 → 生成中 → 待审核 → 审核通过 → 待发布 → 已发布 → 已复盘
- 支持人工修改、驳回、再次生成

### 4. 多平台版本中心

- 同一选题可生成不同平台版本
- 管理平台特有格式、封面比例、标签策略等

### 5. 账号画像中心

- 平台账号配置：
  - 账号定位
  - 目标用户
  - 标题偏好、封面偏好
  - 语气风格、内容边界
  - 禁用表达
  - 历史爆款/低效内容
- Agent生成时自动加载

### 6. 数据复盘中心

- 收集发布数据：
  - 阅读、点赞、评论、收藏、转发
- 分析：
  - 标题、封面、正文、平台、账号、Agent效果
- 输出复盘报告，用于优化下一轮生成

## 四、数据流

```
选题创建
   ↓
Agent Runtime生成内容
   ↓
调用IMA获取参考素材/搜索
   ↓
内容审核
   ↓
同步TurboPush发布
   ↓
各平台发布
   ↓
数据回收
   ↓
生成复盘报告
   ↓
优化Prompt和流程
```

## 五、技术栈（TS 全栈，轻量版）

| 层级          | 技术                                    |
| ------------- | --------------------------------------- |
| 前端          | React + Next.js + TypeScript            |
| 后端 / 中台   | Node.js + NestJS / Express + TypeScript |
| Agent Runtime | Node.js + TypeScript                    |
| 发布底座      | TurboPush (TS)                          |
| 知识/搜索层   | IMA                                     |
| 数据库        | SQLite + Prisma                         |
| 部署          | Docker                                  |

> 注：去掉对象存储、Redis、工作流、向量搜索，SQLite 作为轻量存储即可。

## 七、核心优势

- 基于 TurboPush 已有底座，减少开发量
- SQLite 简单部署，满足 V1 条件
- Agent Runtime 自研，无依赖 Dify/FastGPT
- IMA 提供知识搜索和向量检索
- 支持多账号、多平台、全流程闭环
- 全 TS 全栈，前后端统一语言
