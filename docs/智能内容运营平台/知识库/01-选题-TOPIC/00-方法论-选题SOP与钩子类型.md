# 选题知识库 · 钩子优先的选题 SOP

## 一、选题不是「想写什么」，是「用什么钩子留住人」

2026 年自媒体选题正确流程：

```
找痛点/热点 → 定钩子类型 → 写清前3秒/前3行 → 再展开干货角度
```

**错误流程**：先定标题 → 硬凑内容 → 开头寒暄铺垫（完播/完读必崩）

**AI 效率赛道选题来源**：

1. 近 7 天热词：OpenClaw、Agent、自动化、周报、数字员工…
2. 评论区高频问：「怎么部署」「会不会泄密」「和 ChatGPT 啥区别」
3. 竞品爆款角度：取其钩子结构，换自己的实测细节

---

## 二、选题筛选四问（全过才写）

| 问题                         | 不过则弃         |
| ---------------------------- | ---------------- |
| 3 秒内能说清「跟我有关」吗？ | 太宽、太学术     |
| 有具体痛点或具体结果吗？     | 只有概念没有场景 |
| 能否给出可立刻做的 3 步？    | 纯观点无干货     |
| 我有真实经历/数字吗？        | 纯搬运无信任     |

---

## 三、钩子类型 × 选题角度（AI 赛道示例）

| 钩子类型       | 机制         | 选题示例                       | 开头钩子示例                                   |
| -------------- | ------------ | ------------------------------ | ---------------------------------------------- |
| **结果前置**   | 好奇怎么做到 | AI 把周报从 40 分钟压到 8 分钟 | 「8 分钟搞定周报，因为我让 AI 学了我的 Excel」 |
| **偷懒反差**   | 利益+好奇    | 把烦人工作甩给 AI              | 「我把 3 件破事扔给 AI，自己在\_\_\_」         |
| **反常识**     | 认知冲突     | 别再手把手教 AI 写周报         | 「越教 AI 写周报，你越累」                     |
| **避坑恐吓**   | 损失厌恶     | Agent 安装 3 大坑              | 「第 2 个坑可能泄露你文件」                    |
| **热点借势**   | 从众好奇     | OpenClaw 为什么爆              | 「火不是因为会聊天，是因为会动手」             |
| **身份召唤**   | 精准人群     | 打工人 AI 效率                 | 「打工人别划走：这设置不改等于白装」           |
| **对比逆袭**   | 反差         | 开源 beat 闭源                 | 「Python 项目凭什么比 Rust 王牌更能干活」      |
| **未完成承诺** | 悬念         | 系列第 1 篇                    | 「最关键那步我放评论区/下集」                  |

## 四、爆款选题五要素（缺一不可）

```
痛点选题 + 钩子标题 + 黄金开头 + 干货主体 + 互动钩子
```

**互动钩子要具体**：

- ❌ 「你们觉得呢？」
- ✅ 「你最想自动化的重复工作是什么？评论区点名」
- ✅ 「上面 3 坑你踩过几个？扣数字」

---

## 五、选题卡片模板（入库必填）

```text
【栏目】AI效率 / OpenClaw / 职场自动化
【目标用户】打工人 / 运营 / 程序员小白
【钩子类型】结果前置 / 避坑 / 反常识
【标题候选】（含人群+痛点+数字）
【前3秒/前3行钩子】（30～60字，禁寒暄）
【干货角度】3 个可执行步骤
【证据】真实数字、日期、工具名
【互动钩子】具体评论指令
【valid_until】YYYY-MM-DD（热点必填）
【status】draft / ready
```

## 六、热点选题时效规则

| 类型     | 复审周期 | 处理                |
| -------- | -------- | ------------------- |
| 事件热点 | 7～14 天 | 过期改 `deprecated` |
| 工具爆款 | 30 天    | 更新数据/版本号     |
| 常青干货 | 90 天    | 补充新案例          |

同一热点角度 **最多 3 篇**，第 4 篇必须换钩子类型或人群。

---

## 七、从数据反推选题（复盘沉淀）

高完播/高收藏内容拆回选题库：

1. 保留：**钩子类型 + 标题结构 + 互动问法**
2. 记录：完播率、收藏率、评论热词
3. 衍生：「高完播钩子的变体选题」—— 同钩子，换场景（周报→日报→会议纪要）

## 九、避坑

- 追热点但说不出 **和用户有关** 的一句话 → 不追
- 选题只有「介绍 XX 工具」无痛点 → 改成「解决 XX 烦恼」
- 开头是「最近 XX 很火」→ 改成「火的关键是 **_，跟你有关因为 _**」
- 没有 `valid_until` 的热点选题 → 容易让 Agent 生成过时内容
