# 正文知识库 · 钩子优先的内容结构（完播/完读率导向）

## 一、核心认知：流量不是写出来的，是「留住人」挣来的

各平台算法的第一质量信号：

| 平台       | 核心指标   | 经验门槛                              | 含义               |
| ---------- | ---------- | ------------------------------------- | ------------------ |
| 抖音       | 完播率     | 30% 及格 / 50% 优质 / 70% 爆款潜质    | 多少人看完整条视频 |
| 小红书图文 | 完读率     | 低于 40% 易被减推 / 高于 65% 进优质池 | 用户是否滑到底     |
| 小红书视频 | 播放完成率 | 低于 30% 质量不达标                   | 同完播逻辑         |

**行业共识数据（2025～2026 运营复盘汇总）**：

- 用户平均在 **2.8 秒** 内决定划走还是留下
- **72%** 的用户在第 3 秒前完成「留/走」判断
- 爆款视频 **78%** 在开头 3 秒就抛出核心冲突或结果
- 「大家好 / 今天聊聊」类问候开头，跳出率比「痛点/结果」开头高约 **37%**
- 带悬念结构的笔记，阅读完成率平均比普通笔记高约 **47%**

**写作原则**：先设计钩子链，再填干货。母稿（BODY）也要把「前 3 秒钩子 + 中段节奏 + 结尾承诺」写清楚，方便 REWRITE 按平台压缩。

## 二、万能钩子公式（图文 + 短视频通用）

```
钩子 = 目标用户 + 痛点/利益 + 悬念/反常识
```

拆解：

1. **目标用户**：让用户 0.5 秒内觉得「这跟我有关」（打工人 / 宝妈 / AI 小白…）
2. **痛点/利益**：戳焦虑或给明确收益（省时间 / 避坑 / 涨粉…）
3. **悬念/反常识**：制造信息缺口（「原因竟然是这个」「99% 的人不知道」）

**反例（必删）**：

- 「大家好，今天分享一个 AI 工具」
- 「作为十年运营，我想聊聊…」
- 「这篇文章很有用，建议收藏」（没有具体利益）

**正例（AI 效率赛道）**：

- 「周报还在自己憋？我把重复半年的 Excel 操作交给 AI，8 分钟搞定。」
- 「OpenClaw 爆火不是因为它会聊天，是因为它真的会替你点鼠标。」
- 「面试官最讨厌的 3 句话，你说过几句？」（职场向可迁移）

## 三、六大高完播钩子类型（按场景选用）

| 类型           | 机制               | 模板                             | 适用               |
| -------------- | ------------------ | -------------------------------- | ------------------ |
| **痛点提问**   | 焦虑代入           | 「你是不是也\_\_\_？」           | 教程、避坑         |
| **结果前置**   | 好奇「怎么做到的」 | 「我用***天，把***从 A 拉到 B」  | 测评、方法论       |
| **反常识**     | 认知冲突           | 「别再***了，真相是***」         | 观点、拆解         |
| **恐惧损失**   | 损失厌恶           | 「再不知道***，你会多花***」     | 职场、理财（慎用） |
| **未完成承诺** | 蔡格尼克效应       | 「看到最后告诉你最关键的那一步」 | 中长视频           |
| **冲突对比**   | 反差               | 「别人***，而你***」             | 故事、逆袭         |

**选题时先定钩子类型，再写标题和开头**——不要反过来。

## 四、时间轴结构（短视频口播 / 视频笔记）

参考「电梯时间法则」（碎片注意力模型）：

| 时间段        | 任务         | 必须完成的事                                 |
| ------------- | ------------ | -------------------------------------------- |
| **0～3 秒**   | 生死线       | 痛点 / 结果 / 反常识三选一，禁止寒暄         |
| **3～8 秒**   | 建立相关     | 身份 + 痛点 + 「这跟你有什么关系」           |
| **8～15 秒**  | 第一个干货点 | 给出可感知价值，兑现开头 30% 承诺            |
| **15～30 秒** | 微转折       | 每 12～15 秒：数据对比 / 镜头切换 / 新信息点 |
| **结尾**      | 动作接口     | 具体评论指令 / 关注理由 / 下期悬念           |

**抖音时长建议**：新手优先 **15～40 秒**（完播率友好区间）；干货可 60～90 秒，但每 15 秒必须有「微转折」。

## 五、图文笔记结构（小红书完读率导向）

```
【前 3 行 = 钩子】30～60 字，不点开全文也能看到核心价值
【中段 = 豹肚】每 150～200 字一个小标题；每点 1～3 行
【结尾 = 豹尾】总结一句 + 具体互动问题 + 自然收藏理由
```

**中段节奏钩子**（防止滑走）：

- 第 2 屏：「很多人卡在这里，其实第二步才是关键」
- 配图悬念：「关键对比在第 3 张图」
- 清单/checklist：让用户觉得「不收藏亏了」

**收藏率经验值**：收藏率 **≥3%** 有机会进扩量池；**≥8%** 有流量跃迁可能。收藏动机 = 可复用（清单、模板、步骤、避坑表）。

## 六、母稿五段式（给 BODY Agent 的标准输出）

```text
1. 【钩子段】50～80 字：用户+痛点+悬念，可直接作平台开头
2. 【共鸣段】80～150 字：场景细节，证明「我懂你」
3. 【干货段】3～5 点，每点：小标题 + 2～4 行 + 可量化结果
4. 【证据段】真实数字、前后对比、一次具体经历（禁虚构）
5. 【收尾段】一句总结 + 互动钩子（具体问题，非「你怎么看」）
```

**母稿禁止**：平台专属话术（「点赞收藏」）、空泛形容词（「非常重要」「干货满满」）。

## 八、复盘指标（发布后必看）

| 指标              | 行动                               |
| ----------------- | ---------------------------------- |
| 前 3 秒流失 > 40% | **只改开头**，不要动全文           |
| 中段某秒大量跳出  | 在该时间点前插入「微转折」或删废话 |
| 完播高、互动低    | 结尾改成具体评论指令               |
| 点击高、完读低    | 标题/封面过度承诺，收紧开头预期    |

**A/B 测试**：同一内容准备 2 个开头（提问式 vs 结果前置），保留完播率高 **30%+** 的版本。

## 九、避坑清单

- 标题/封面承诺 > 前 3 秒交付 → 「后悔阈值」过大，秒划走
- 开头解释背景、自我介绍、铺垫环境 → 一律后置或删除
- 全文一个节奏无转折 → 30 秒后完播断崖
- 互动问「大家觉得呢」→ 太泛，评论率差
- 虚构数据、绝对化收益 → 限流 + 信任崩塌
