# 正文知识库 · 小红书钩子与完读率写法

## 一、平台差异：小红书要的是「搜得到 + 读得完 + 舍得藏」

- **搜索流量占比高**（约 40～60%）：标题和正文前几行要带 **场景词 + 痛点词**（如「打工人 周报 AI 自动化」）
- **双列信息流**：封面决定点击，**前 3 行决定完读**
- **收藏权重高**：收藏 =「以后用得上」；收藏率 3% 以上才有机会扩量

REWRITE 产出小红书版时：**钩子前置、短段、可收藏、可搜索**。

---

## 二、前 3 行生死线（30～60 字）

用户不点开「全文」也必须看到钩子。禁用「自嗨式开头」：

| 类型     | 反例                           | 正例                                    |
| -------- | ------------------------------ | --------------------------------------- |
| 寒暄     | 「今天来分享一个超好用的工具」 | 「写周报写到崩溃的姐妹，先别划走」      |
| 自我介绍 | 「我是做了 3 年运营的 xxx」    | 「我用 AI 把周报从 40 分钟压到 8 分钟」 |
| 空洞干货 | 「分享几个 AI 技巧」           | 「这 3 个设置不改，你的 AI 永远像智障」 |

### 三大爆款开头公式（三选一）

**1. 痛点共鸣式**

> 模板：具体场景 + 情绪词 + 暗示有解  
> 例：「夏天油皮化妆 2 小时就花？你不是手法问题，是第一步产品选错了。」

**2. 惊人结论式（结果前置）**

> 模板：数字 + 时间 + 可感知变化  
> 例：「坦白局：我只改了 Excel 里 1 个重复操作，AI 就自动帮我跑周报了。」

**3. 悬念反常识式**

> 模板：打破常识 + 信息缺口  
> 例：「OpenClaw 爆火，不是因为会聊天，是因为它真的会替你点鼠标。」

**AI 工具赛道实测向钩子**：

```text
❌ 推荐一个 AI 神器（太泛）
✅ 我把周报扔给 AI 后，它学会了我 Excel 里的骚操作（反差+好奇）

❌ AI 办公效率提升（无场景）
✅ 打工人别划走：这 3 个 AI 设置不改，等于白装（身份+恐吓）

❌ OpenClaw 使用教程（无利益）
✅ 5 块钱/月养「数字同事」，微信里一句话让它帮我整理文件（利益+具体）
```

## 三、正文结构：虎头 · 豹肚 · 豹尾

### 虎头（前 3 行）

- 戳痛点 + 给承诺 + 显价值（「这篇给你 3 个能立刻试的步骤」）
- 控制在 **50～80 字**，手机 **不超过 4 行**

### 豹肚（主体）

| 规则     | 说明                                        |
| -------- | ------------------------------------------- |
| 总字数   | 干货 **600～1200 字**；清单类可更短但要密   |
| 分段     | 每段 **1～3 行**，绝不大段文字墙            |
| 小标题   | 每 **150～200 字** 一个 ✅/❶ 小标题，拉滑动 |
| emoji    | 全文 **5～12 个**，点缀不堆砌               |
| 信息甜点 | 每屏一个干货点、金句或对比                  |

**五段式节奏（高收藏干货）**：

```text
钩子（痛点/结论）
↓
放大痛点（❌ 踩坑 1/2/3，每条一行）
↓
解决方案（✅ 步骤 1/2/3，每条可立刻做）
↓
效果可视化（前后对比、具体时间数字）
↓
行动指令（收藏理由 + 具体评论问题）
```

### 豹尾（结尾）

- **总结一句**：「所以记住：开头钩子 > 工具本身」
- **收藏钩子**（自然融入，勿单独一行硬广）：「建议收藏，下次做周报直接翻第三步」
- **具体互动**（禁「你怎么看」）：
  - 「你最难忍的重复工作是什么？评论区说，我下期按票做」
  - 「上面 3 招你用过哪招？没用过的扣 1」

## 四、完读率专用「中段钩子」

防止读者滑到一半流失：

1. **预告式**：「很多人栽在第 2 步，其实关键在/settings/」
2. **对比式**：「左边是普通人用法，右边是这样用」
3. **图悬念**：「对比图放第 3 张，差别一眼看懂」
4. **系列预告**：「下篇拆安装避坑，明天更」

带悬念结构的笔记，完成率平均比普通笔记 **高约 47%**（运营复盘经验值）。

---

## 五、搜索 + 收藏双优化

**标题/正文关键词**：前 18 字含 2 个核心词；正文每 300 字自然重复 1 次目标词。

公式：**【人群】+【痛点】+【解决方案】+【证明】**

例：`打工人 AI 周报｜8 分钟搞定｜我实测 2 周`

**高收藏内容形态**（优先用）：

- 一张清单搞定（附步骤编号）
- 避坑 ❌ 列表
- Before / After 对比
- 可复用模板（口令、话术、检查表）

## 六、完整示例（AI 周报 · 可直接作 REWRITE 参考）

```text
写周报写到想辞职的，先别划走 🙋

你不是不会写，是每天都在重复同样的 Excel 操作。
我上周把这事交给 AI，40 分钟 → 8 分钟，而且它把我操作录成了「技能」。

❌ 踩坑 1：只让 AI 写文字，不告诉它你的表格结构
❌ 踩坑 2：每次从零描述背景，AI 像失忆
❌ 踩坑 3：不会存技能，下次还得重教

✅ 步骤 1：先做一次周报，让 AI 看着你操作（录屏/分步描述）
✅ 步骤 2：做完问它「要不要存成技能」—— 选是
✅ 步骤 3：下次只说「按上周技能跑周报」，别重新讲一遍

📊 我的变化：2 周省了约 6 小时，出错率反而更低

💡 记住：AI 省时间的关键不是「会写」，是「会学你的重复劳动」

👇 你最想自动化哪项破事？评论区点名，我按票做下一篇
（做周报的姐妹建议收藏，第三步最容易忘）
```

## 七、数据复盘与改写优先级

| 信号           | 改什么                          |
| -------------- | ------------------------------- |
| 点击低         | 改封面 + 标题关键词，不是改正文 |
| 点击高、完读低 | 改前 3 行钩子，是否标题党       |
| 完读高、收藏低 | 加清单/模板/避坑表，补收藏钩子  |
| 收藏高、评论少 | 结尾问题改具体、可站队          |

---

## 八、避坑

- 外链、微信号硬导 → 限流
- 「最」「第一」「100%」→ 违禁风险
- 收藏引导单独成段「请收藏」→ 易被判定诱导
- 开头铺垫过长 → 3 屏内无信息增量 = 无效阅读
- 干货只说「要多用 AI」→ 不可操作 = 不收藏
