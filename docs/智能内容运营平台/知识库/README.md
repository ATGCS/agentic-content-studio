# ACS 知识库 · 本地源文件规划

本目录是 **Agentic Content Studio** 知识库的「源文件仓库」：在这里按类型编写、重构 Markdown，再导入到系统本地知识库（或同步到 IMA）。

## 内容写作原则（2026 重构）

**一切以钩子 + 完播/完读率为导向**，不是写「教科书式规范」：

1. 先定钩子类型，再写标题和开头
2. 前 3 秒 / 前 3 行决定流量，中段靠「微转折」拉留存
3. 数据来自平台运营复盘与爆款结构拆解，需定期更新热点案例
4. 每篇文档要有 **可复制句式 + 完整示例 + 复盘指标**

## 文档命名规范

```
{序号}-{分类}-{简短描述}.md
```

- `00-` 开头：方法论 / 总览 / SOP
- `01-`、`02-`：垂直领域或平台拆分
- `_INDEX.md`：该库目录索引（给人看，不导入）
- `_模板-*.md`：写作模板（不导入）

## 单篇文档规范

每篇可检索文档建议包含 YAML 头（见 `_模板-单篇文档.md`）：

- `kb_type`：与目录后缀一致
- `status`：`draft` | `ready` | `deprecated`
- `keywords`：便于人工检索

正文结构建议：**目标 → 适用场景 → 规则/公式 → 示例 → 避坑**。

## 导入到 ACS

在项目根目录执行：

```bash
pnpm kb:import
```

脚本会：

1. 按子目录创建/更新 **本地知识库**（`source: local`）
2. 将各目录下 `.md` 文档 upsert 到对应库（跳过 `_` 开头文件）
3. 自动设置 `agentType` 绑定

导入后在 **设置 → IMA 知识库** 检查绑定，即可用于 AI 生成检索。

## 与 IMA 远程库的关系

- **本地目录**：版本管理、协作编辑、重构的主战场
- **IMA 远程**：若已配置 IMA，可在远程建同名库并上传；系统优先检索已启用且文档已同步的库
- **推荐做法**：先在本地写好 → `pnpm kb:import` 验证效果 → 稳定后再同步到 IMA
