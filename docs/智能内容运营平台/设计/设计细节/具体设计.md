# 具体设计

## 13. 推荐业务流程

```text
创建选题
  ↓
选择目标平台与平台账号
  ↓
平台内置 Agent 读取任务与知识库
  ↓
生成标题/封面/正文/图片/标签
  ↓
生成结果写入内容资产库
  ↓
内容状态进入待审核
  ↓
运营人员审核/修改/驳回
  ↓
审核通过
  ↓
生成平台账号版本
  ↓
同步草稿/发布/半自动发布
  ↓
回收运营数据
  ↓
数据复盘 Agent 生成复盘结论
  ↓
沉淀回知识库
  ↓
优化下一次生成
```

### 12.1 外部 Agent 接入流程

```text
外部桌面 Agent
  ↓
调用平台任务接口
  ↓
读取知识库和任务信息
  ↓
生成内容
  ↓
调用内容写入接口
  ↓
平台统一进入待审核
  ↓
后续流程与内部 Agent 一致
```

## 14. 状态机设计

```text
待创建
  ↓
待生成
  ↓
生成中
  ↓
待审核
  ├── 驳回 → 重新生成 / 人工修改
  └── 通过
        ↓
      待发布
        ↓
      发布中
        ↓
      已发布
        ↓
      数据回收中
        ↓
      已复盘
```

### 13.1 审核维度

| 审核项       | 说明                                   |
| ------------ | -------------------------------------- |
| 标题审核     | 是否符合平台调性，是否标题党，是否违规 |
| 封面审核     | 是否清晰，是否符合比例，是否有版权风险 |
| 正文审核     | 是否准确、完整、有价值                 |
| 图片审核     | 是否侵权、是否违规、是否清晰           |
| 标签审核     | 是否符合平台规则                       |
| 合规审核     | 敏感词、夸大宣传、医疗金融等风险       |
| AIGC 审核    | 是否需要标识，是否已声明               |
| 平台适配审核 | 是否符合目标平台格式                   |

## 15. MVP 第一版建议

### 14.1 MVP 目标

第一版不追求全平台发布，而是验证核心闭环：

> 平台创建选题 → 选择目标平台与平台账号 → 内置 Agent 读取对应账号知识库 → Agent 生成内容 → 写入内容资产库 → 内容进入待审核 → 人工审核通过 → 形成可发布内容包。

同时保留外部 Agent 接口能力：

> 外部 Agent 也可以通过标准接口读取任务、读取知识库、写入生成结果，但不作为主流程依赖。

### 14.2 P0 功能

| 功能               | 说明                                                         |
| ------------------ | ------------------------------------------------------------ |
| 内容项目管理       | 创建和管理选题、文章、平台账号版本                           |
| 平台账号管理       | 管理公众号、小红书、抖音等平台的多个账号、授权状态、账号定位和权限 |
| 知识库管理         | 标题、封面、正文、平台规则分类                               |
| 内置标题生成 Agent | 基于标题知识库生成多个标题候选                               |
| 内置标签生成 Agent | 根据平台规则生成标签和关键词                                 |
| 内置平台改写 Agent | 生成公众号/小红书/抖音等平台版本                             |
| 审核辅助 Agent     | 检查敏感词、AIGC 标识、平台规则                              |
| 待审核状态         | Agent 生成后默认待审核                                       |
| 审核通过/驳回      | 支持人工确认和修改意见                                       |
| 基础素材管理       | 上传、绑定封面和正文图片                                     |
| 外部 Agent 接口    | 支持外部 Agent 写入内容，作为兼容能力                        |

### 14.3 P1 功能

| 功能           | 说明                                         |
| -------------- | -------------------------------------------- |
| 正文生成 Agent | 基于知识库生成正文内容                       |
| 封面文案 Agent | 生成封面标题、副标题、利益点                 |
| 选题 Agent     | 根据行业资料、热点和历史数据生成选题         |
| 公众号草稿同步 | 按已绑定公众号账号同步到对应草稿箱           |
| 基础数据录入   | 支持人工录入或导入数据                       |
| 数据看板       | 查看阅读、点赞、收藏、评论等指标             |
| 知识库引用记录 | 记录每次生成调用了哪些知识点                 |
| 生成记录管理   | 记录 Agent、模型、提示词版本、时间、输出内容 |

### 14.4 P2 功能

| 功能               | 说明                                      |
| ------------------ | ----------------------------------------- |
| 图片生成 Agent     | 生成封面图或正文配图                      |
| 视频脚本 Agent     | 生成短视频脚本、分镜、口播稿              |
| 抖音授权发布       | 支持用户授权后的内容发布流程              |
| 小红书半自动发布包 | 生成复制友好的内容包                      |
| 数据复盘 Agent     | 根据数据生成复盘结论                      |
| 知识库效果分析     | 评估知识点对内容表现的贡献                |
| Agent 质量评分     | 分析不同 Agent 生成内容的审核通过率和表现 |

### 14.5 第一版不建议做

| 功能                | 原因                               |
| ------------------- | ---------------------------------- |
| 全平台自动发布      | 平台接口、权限和风控不确定         |
| 自动视频生成全链路  | 复杂度高，适合后期                 |
| 小红书全自动发布    | 官方开放边界不适合作为第一阶段承诺 |
| 抖音后台静默发布    | 抖音更适合用户授权/主动触发场景    |
| 复杂 BI 系统        | 数据积累不足前价值有限             |
| 通用 Agent 搭建平台 | 容易偏离内容运营主线               |

## 18. 数据复盘设计

### 17.1 数据指标

| 维度     | 指标                                           |
| -------- | ---------------------------------------------- |
| 曝光     | 曝光量、展现量、推荐量                         |
| 点击     | 阅读量、点击率、封面点击                       |
| 互动     | 点赞、收藏、评论、转发                         |
| 留存     | 完读率、完播率、平均观看时长                   |
| 转化     | 私信、关注、表单、商品点击、咨询               |
| 内容     | 选题类型、标题类型、封面类型、正文结构         |
| 知识库   | 调用了哪些知识点，效果如何                     |
| Agent    | 哪个 Agent 生成，审核通过率如何                |
| 平台账号 | 哪个账号发布，账号粉丝、内容表现、矩阵对比如何 |

### 17.2 复盘目标

数据复盘不应只看总阅读量，而应回答：

- 哪类选题更容易获得曝光？
- 哪类标题点击率更高？
- 哪类封面更容易带来收藏？
- 哪种正文结构更容易提高完读率？
- 哪个平台更适合某类选题？
- 哪个账号更适合某类内容？
- 同一平台不同账号的内容表现差异如何？
- 哪个 Agent 生成质量更稳定？
- 哪些知识库规则真正有效？
- 哪些提示词版本效果更好？

## 20. 合规与风险分析

### 19.1 AIGC 标识风险

《人工智能生成合成内容标识办法》明确指出，人工智能生成合成内容包括文本、图片、音频、视频、虚拟场景等信息；标识包括显式标识和隐式标识；用户发布生成合成内容时应主动声明并使用服务提供者提供的标识功能。该办法自 2025 年 9 月 1 日起施行。

因此系统必须内置相关字段：

| 字段              | 说明                      |
| ----------------- | ------------------------- |
| is_ai_generated   | 是否 AI 生成              |
| ai_content_type   | 文本 / 图片 / 视频 / 混合 |
| ai_label_required | 是否需要 AIGC 标识        |
| ai_label_status   | 未标识 / 已标识 / 不适用  |
| agent_type        | internal / external       |
| generation_agent  | 生成 Agent                |
| generation_model  | 生成模型                  |
| generation_time   | 生成时间                  |
| human_reviewed    | 是否人工审核              |
| risk_check_result | 风险检测结果              |

### 19.2 Agent 执行风险

风险点：

- Agent 生成内容不稳定；
- 多 Agent 输出风格不一致；
- 提示词版本混乱；
- 模型调用成本不可控；
- 外部 Agent 写入内容质量不可控。

应对策略：

- 内置 Agent 优先；
- 所有 Agent 输出默认待审核；
- 建立 Agent 配置管理；
- 建立提示词版本管理；
- 建立生成日志；
- 建立成本监控；
- 外部 Agent 限制权限和写入范围。

### 19.3 平台账号与授权风险

风险点：

- 多平台账号授权状态不同；
- 不同账号权限、角色、品牌归属不清；
- 员工离职或角色变化后账号权限未及时回收；
- 内容误发到错误账号；
- 数据回收无法准确归因到具体账号。

应对策略：

- 平台账号统一管理；
- 每个账号绑定负责人、品牌、平台、授权状态；
- 内容发布前必须确认目标账号；
- 审核权限、发布权限按账号配置；
- 数据回收结果必须绑定平台账号 ID；
- 关键发布动作保留操作日志。

### 19.4 平台接口风险

风险点：

- 平台接口开放程度不同；
- 发布能力可能需要授权；
- 数据能力可能需要账号授权；
- 平台规则可能变化；
- 非官方接口存在封号和失效风险。

应对策略：

- 官方 API 优先；
- 不使用非官方爬虫发布；
- 不承诺所有平台自动发布；
- 每个平台单独设计接入方式；
- 支持自动发布、授权发布、半自动发布三种模式。

### 19.5 内容质量风险

风险点：

- 事实错误；
- 夸大宣传；
- 侵权图片；
- 标题党；
- 平台禁词；
- 不符合账号调性；
- 品牌口径冲突。

应对策略：

- 所有 AI 写入默认待审核；
- 建立敏感词库；
- 建立事实核查机制；
- 建立图片版权字段；
- 建立账号风格规则；
- 建立人工审核流程。

## 21. 阶段规划

### 20.1 第一阶段：内容中台 + 基础内置 Agent MVP

目标：跑通平台内部 Agent 生成、写入和人工审核。

核心能力：

- 内容项目管理；
- 平台账号管理；
- 知识库分类；
- 内置标题生成 Agent；
- 内置标签生成 Agent；
- 内置平台改写 Agent；
- 审核辅助 Agent；
- 待审核状态；
- 人工审核；
- 平台账号版本管理；
- 基础素材管理；
- 外部 Agent 兼容接口。

成功标准：

- 平台内置 Agent 能稳定生成标题、标签和平台改写版本；
- 运营人员能在平台审核；
- 每个内容项目都有完整资产；
- 每个内容版本都能绑定目标平台账号；
- 内容状态清晰可追踪；
- 外部 Agent 可作为补充写入内容。

### 20.2 第二阶段：公众号深度对接 + 正文/封面 Agent

目标：验证官方平台发布链路，并增强内容生成能力。

核心能力：

- 正文生成 Agent；
- 封面文案 Agent；
- 公众号账号授权管理；
- 公众号素材上传；
- 按账号同步公众号草稿；
- 公众号发布任务；
- 公众号数据回收；
- 公众号版本复盘。

成功标准：

- 平台内容可以同步到公众号草稿；
- 审核通过后可以进入发布流程；
- 发布后能看到基础数据；
- 能按标题、封面、正文结构做复盘。

### 20.3 第三阶段：小红书 / 抖音平台适配 + 数据复盘 Agent

目标：实现多平台内容版本管理和半自动/授权发布。

核心能力：

- 小红书账号版本管理；
- 小红书图文笔记版本；
- 小红书封面与标签规则；
- 抖音账号版本管理；
- 抖音标题、话题、脚本、封面管理；
- 抖音授权发布或半自动发布；
- 抖音视频数据回收；
- 数据复盘 Agent；
- 数据对比看板。

成功标准：

- 同一个选题可以生成多平台版本；
- 运营人员不用重复改格式；
- 发布数据能统一沉淀；
- 能分析不同平台内容表现差异。

### 20.4 第四阶段：Agent 编排与知识库自动优化

目标：让数据反哺知识库和 Agent。

核心能力：

- 选题 Agent；
- 图片生成 Agent；
- 视频脚本 Agent；
- 发布策略 Agent；
- 爆款知识自动沉淀；
- 标题模板胜率；
- 封面类型效果；
- 平台内容适配建议；
- Agent 生成质量评估；
- 自动复盘报告。

成功标准：

- 知识库不只是存资料，而是能被数据验证；
- 后续生成质量明显提升；
- 团队可以根据数据优化内容策略；
- 平台可以自动推荐下一轮内容策略。

## 22. 商业化方向

### 21.1 自媒体团队 / MCN

痛点：

- 多账号管理混乱；
- 内容生产量大；
- 多平台重复发布；
- 爆款经验靠个人；
- 数据复盘难。

卖点：

- 内置 Agent 提升生产效率；
- 管理内容资产；
- 支持多平台版本；
- 沉淀爆款方法论。

### 21.2 品牌内容团队

痛点：

- 品牌口径要统一；
- 内容需要审核；
- 不能完全交给 AI；
- 多平台运营成本高；
- 需要看转化效果。

卖点：

- 品牌知识库；
- 内置 Agent 可控生成；
- 审核流；
- 合规记录；
- 多平台内容适配；
- 数据复盘。

### 21.3 企业内部内容中台

痛点：

- 市场、运营、销售、品牌内容分散；
- 历史资料无法复用；
- AI 生成不可控；
- 内容审核链路复杂。

卖点：

- 企业知识库；
- 内置 Agent 执行；
- AI 生成留痕；
- 内容流程管理；
- 权限控制；
- 合规审计。

## 23. 最终建议

本项目具备可行性，但第一阶段必须收窄边界。

推荐路线：

1. 先做内容资产管理、平台账号管理、知识库、基础内置 Agent、审核流；
2. 同时保留外部 Agent 标准接入能力；
3. 再优先深接微信公众号；
4. 然后做小红书、抖音的平台版本适配和半自动/授权发布；
5. 最后做 Agent 编排、数据复盘和知识库自动优化。

### 22.1 核心壁垒

| 壁垒           | 说明                                                     |
| -------------- | -------------------------------------------------------- |
| 知识库结构化   | 把标题、封面、正文、平台规则拆成可调用知识点             |
| Agent 内置化   | 核心生成任务由平台统一执行和管理                         |
| 外部兼容化     | 允许外部桌面 Agent 通过标准协议接入                      |
| 内容资产化     | 每个选题、版本、素材、状态都可管理                       |
| 多账号矩阵化   | 每个内容项目、平台版本、发布任务和数据结果都绑定平台账号 |
| 审核流程化     | AI 生成内容默认待审核                                    |
| 多平台适配化   | 一题多版，而不是一稿多发                                 |
| 数据复盘闭环化 | 发布数据反哺知识库和 Agent                               |

### 22.2 最终结论

本项目不应做成普通 AI 写作工具，也不应做成普通多平台发布工具，更不应只做外部 Agent 的接口后台。

最优定位是：

> **内置 Agent 的智能内容运营中台。**

平台应以内部 Agent 为核心能力，以外部 Agent 为兼容扩展，以平台账号管理、知识库、内容资产、审核流、多平台适配和数据复盘为长期壁垒。

项目核心价值是：

> 让内容生产从一次性人工创作，变成可管理、可追踪、可审核、可复盘、可持续优化的智能运营流水线。
