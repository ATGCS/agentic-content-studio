# 开发规范文档 V1

## 一、Monorepo 结构

仓库：[ATGCS/agentic-content-studio](https://github.com/ATGCS/agentic-content-studio)

使用 **pnpm workspace**（或 npm workspaces）。

```text
apps/
  web/          # Next.js 15+，App Router
  api/          # Fastify 或 NestJS（二选一，团队定稿）

packages/
  db/           # Prisma schema、migrations、seed
  core/         # 共享类型、枚举、错误码、工具
  ai-runtime/
  content-center/
  review-center/
  analytics-center/
  account-profile/
  turbopush-adapter/
  ima-provider/
```

`apps/*` 可依赖 `packages/*`；`packages/*` 之间禁止循环依赖（见模块依赖图）。

## 二、TypeScript

- `strict: true`
- 禁止 `any`（除第三方 raw 响应需 `unknown` + 校验）
- 公共类型放 `packages/core`，API DTO 与 Prisma 类型分离

## 三、ESLint / Prettier

- 根目录统一 `eslint.config.js` + `.prettierrc`
- 提交前：`pnpm lint && pnpm typecheck`
- 推荐规则：`@typescript-eslint/recommended`、`simple-import-sort`

## 四、API 规范

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 路由

- 前缀 `/api`
- REST 资源名复数：`/contents`、`/reviews`
- 动作型：`POST /reviews/:id/approve`

### 校验

- 入参使用 zod，在 handler 边界校验
- 错误码见《API设计文档 V1》第十八节

## 五、数据库

- 仅 `packages/db` 可 `prisma migrate`
- 业务代码通过 `@repo/db` 导出 `prisma` 单例
- 禁止在 Controller 写原始 SQL（复盘统计除外需评审）

## 六、Git 分支

| 分支 | 用途 |
| ---- | ---- |
| main | 可发布 |
| develop | 集成 |
| feature/* | 功能 |
| fix/* | 修复 |

PR 需：描述 + 自测说明；关联文档变更时更新 `docs/智能内容运营平台/`。

## 七、环境变量

| 变量 | 说明 |
| ---- | ---- |
| DATABASE_URL | SQLite 路径 |
| JWT_SECRET | 认证 |
| DEEPSEEK_API_KEY | 模型 |
| TURBOPUSH_BASE_URL | Adapter |
| IMA_API_KEY | 搜索 |

禁止提交 `.env`；提供 `.env.example`。

## 八、日志

- 结构化 JSON：`level`, `msg`, `requestId`, `userId`
- Agent 执行：关联 `agentRunId`
