# 运营流程与模块协作文档

## 1. 平台定位

智能内容运营平台围绕“多平台账号 + 内容项目 + AI Agent + 审核发布 + 数据复盘”组织业务流程，目标是把内容运营从手工分散操作，收敛为一条可追踪、可审核、可复盘的流水线。

核心闭环：

```text
工作台总览
  ↓
平台账号配置 / 知识库配置 / 素材沉淀
  ↓
选题创建
  ↓
内容项目生产
  ↓
Agent 生成与改写
  ↓
审核中心审批
  ↓
发布管理执行
  ↓
数据复盘沉淀
  ↓
反哺选题、账号画像、知识库、Prompt
```

## 2. 模块职责

### 2.1 工作台

入口页面，用于让运营人员快速判断当天工作重点。

职责：

- 展示内容项目、审核、发布、数据等关键指标。
- 汇总待处理事项，例如待审核、待发布、失败任务、异常账号。
- 作为运营人员进入具体模块的导航入口。

对应页面：

- `/dashboard`

相关接口：

- `GET /api/dashboard/stats`

### 2.2 平台账号

管理各内容平台账号授权、账号画像、账号负责人和可用能力。

职责：

- 同步或维护公众号、小红书、抖音、视频号、B站、知乎等账号。
- 维护账号定位、内容风格、目标人群、账号负责人。
- 作为内容生成、审核和发布时的平台与账号选择来源。

对应页面：

- `/accounts`
- `/accounts/[id]`

相关接口：

- `GET /api/accounts`
- `GET /api/accounts/:id`
- `POST /api/accounts/sync`
- `GET /api/account-profiles`
- `PUT /api/account-profiles/:accountId`

与其他模块关系：

- 内容项目选择目标平台和目标账号时依赖平台账号。
- Agent 生成时根据账号画像调整语气、风格和平台表达。
- 发布管理创建发布任务时依赖授权状态正常的账号。
- 数据复盘按账号维度汇总表现，并反哺账号画像。

### 2.3 知识库

配置外部 IMA 知识库，给 Agent 提供检索增强上下文。

职责：

- 配置 IMA 连接信息。
- 同步知识库列表。
- 启用、禁用或设置默认知识库。
- 在内容生成时提供行业资料、品牌资料、活动资料、历史案例。

对应页面：

- `/settings/ima`
- `/knowledge`
- `/knowledge/[id]`

相关接口：

- `GET /api/ima/config`
- `PUT /api/ima/config`
- `GET /api/ima/knowledge-bases`
- `POST /api/ima/knowledge-bases/sync`
- `PATCH /api/ima/knowledge-bases/:id`
- `POST /api/ima/search`

与其他模块关系：

- Agent 生成工作台调用知识库，提高内容准确性和贴合度。
- 选题与内容项目可关联知识库检索结果。
- 数据复盘中的成功案例可以沉淀回知识库。

### 2.4 素材库

统一管理图片、视频、音频、附件等素材。

职责：

- 保存内容项目可复用的封面、正文配图、视频素材、附件。
- 支持按素材类型、用途和来源筛选。
- 为发布包生成和内容详情提供素材来源。

对应页面：

- `/materials`

相关接口：

- `GET /api/materials`
- `GET /api/materials/:id`
- `PATCH /api/materials/:id`
- `DELETE /api/materials/:id`
- `GET /api/contents/:id/materials`
- `POST /api/contents/:id/materials`

与其他模块关系：

- 内容项目绑定素材。
- 发布管理生成发布包时读取素材。
- 审核中心检查素材是否合规。

### 2.5 选题管理

沉淀和管理运营选题，是内容项目的上游来源。

职责：

- 创建选题，维护选题描述、来源、目标平台。
- 管理选题状态，从待生产推进到内容项目。
- 支持运营团队统一规划内容方向。

对应页面：

- `/topics`

相关接口：

- `GET /api/topics`
- `POST /api/topics`
- `GET /api/topics/:id`
- `PATCH /api/topics/:id`
- `DELETE /api/topics/:id`

与其他模块关系：

- 内容项目可以从选题创建。
- Agent 生成可读取选题描述作为生成输入。
- 数据复盘结果可反向影响后续选题。

### 2.6 内容项目

承载一篇或一组内容的核心业务对象。

职责：

- 管理内容标题、摘要、状态、负责人、关联选题。
- 管理多平台版本，例如公众号版、小红书版、抖音脚本版。
- 作为 Agent 生成、审核、发布和数据复盘的主线实体。

对应页面：

- `/contents`
- `/contents/[id]`

相关接口：

- `GET /api/contents`
- `POST /api/contents`
- `GET /api/contents/:id`
- `PATCH /api/contents/:id`
- `POST /api/contents/:id/generate`
- `GET /api/contents/:id/versions`
- `POST /api/contents/:id/versions/generate`
- `PATCH /api/versions/:versionId`

与其他模块关系：

- 从选题生成内容项目。
- 调用 Agent 生成标题、正文、平台改写、标签、封面文案。
- 内容版本提交审核。
- 审核通过后创建发布任务。
- 发布后进入数据复盘。

### 2.7 Agent 任务

跟踪所有 AI Agent 的执行记录、状态、成本和结果。

职责：

- 展示待执行、执行中、已完成、失败、待审核、已写入内容等任务状态。
- 按任务类型、Agent 类型、状态、时间筛选。
- 记录模型、知识库、耗时、成本、目标平台和目标账号。
- 作为排查 AI 生成链路的运维视图。

对应页面：

- `/agent-tasks`
- `/agent-tasks/[id]`
- `/ai-generate`

相关接口：

- `POST /api/agents/title/run`
- `POST /api/agents/rewrite/run`
- `POST /api/agents/review/run`
- `GET /api/agent-runs`
- `GET /api/agents`

与其他模块关系：

- 读取内容项目、目标平台、目标账号和知识库。
- 生成结果写入内容版本。
- 审核辅助 Agent 可自动生成风险提示。
- 数据复盘 Agent 可生成复盘报告。

### 2.8 Prompt 管理

维护不同 Agent 的 Prompt 模板。

职责：

- 管理标题生成、正文生成、平台改写、审核检查等 Prompt。
- 支持版本管理。
- 为 Agent Runtime 提供提示词模板。

对应页面：

- `/prompts`

相关接口：

- `GET /api/prompts`
- `POST /api/prompts`
- `PATCH /api/prompts/:id`

与其他模块关系：

- Agent 任务执行时读取 Prompt。
- 数据复盘可反向指导 Prompt 优化。

### 2.9 审核中心

承接内容版本的人工或 AI 辅助审核。

职责：

- 展示待审核、已通过、已驳回、高风险内容。
- 查看内容风险点、审核建议、提交人和来源。
- 支持通过、驳回和填写驳回原因。
- 只有审核通过的内容才能进入发布准备。

对应页面：

- `/reviews`
- `/reviews/[id]`

相关接口：

- `GET /api/reviews`
- `GET /api/reviews/:id`
- `POST /api/reviews`
- `POST /api/reviews/:id/approve`
- `POST /api/reviews/:id/reject`

与其他模块关系：

- 内容项目提交审核后生成审核任务。
- 审核辅助 Agent 可提供风险识别结果。
- 审核通过后进入发布管理。
- 审核驳回后回到内容项目或 Agent 重新生成。

### 2.10 发布管理

负责把审核通过的内容按平台和账号生成发布任务并执行。

职责：

- 管理待发布内容。
- 管理公众号草稿同步、小红书发布包、抖音发布包等平台发布准备。
- 创建定时或立即发布任务。
- 执行发布、取消发布、查看发布记录。

对应页面：

- `/publishing`

相关接口：

- `GET /api/publishing/tasks`
- `POST /api/publishing/tasks`
- `GET /api/publishing/tasks/:id`
- `POST /api/publishing/tasks/:id/publish`
- `POST /api/publishing/tasks/:id/cancel`

与其他模块关系：

- 只接收审核通过的内容版本。
- 依赖平台账号授权状态。
- 发布完成后生成发布记录，并进入数据复盘。

### 2.11 数据复盘

对已发布内容进行数据同步、分析和报告生成。

职责：

- 同步阅读、播放、互动、转化等发布数据。
- 按内容、平台、账号、选题分析表现。
- 生成内容复盘报告。
- 提炼高表现内容模式，反哺选题、知识库和 Prompt。

对应页面：

- `/analytics`

相关接口：

- `POST /api/analytics/sync`
- `GET /api/analytics/contents/:contentId`
- `POST /api/analytics/reports/generate`
- `GET /api/analytics/reports/:id`
- `GET /api/analytics/reports`

与其他模块关系：

- 从发布管理获取发布记录。
- 复盘结论反哺内容项目、选题、账号画像、知识库和 Prompt。

### 2.12 系统设置

管理平台级配置。

职责：

- 配置 IMA、模型、系统参数、权限策略等。
- 支撑业务模块正常运行。

对应页面：

- `/settings`
- `/settings/ima`

## 3. 标准运营流程

### 阶段一：基础配置

1. 管理员登录平台。
2. 在平台账号中同步或新增各平台账号。
3. 给账号维护负责人、账号定位和内容风格。
4. 在知识库中配置 IMA 并同步知识库。
5. 在素材库中维护品牌素材、产品素材、历史案例。
6. 在 Prompt 管理中维护各类 Agent Prompt。

产出：

- 可用账号池。
- 可用知识库。
- 可用素材库。
- 可执行 Agent 配置。

### 阶段二：选题策划

1. 运营人员在选题管理中创建选题。
2. 填写选题标题、说明、来源和目标平台。
3. 选题通过内部确认后，进入内容生产。

产出：

- 选题记录。
- 目标平台建议。
- 内容项目创建依据。

### 阶段三：内容项目创建

1. 从选题创建内容项目，或直接新建内容项目。
2. 选择目标平台和目标账号。
3. 绑定素材、知识库和负责人。
4. 进入 Agent 内容生成工作台。

产出：

- 内容项目。
- 初始内容版本。
- 目标平台和账号关系。

### 阶段四：Agent 生成与改写

1. 在 Agent 内容生成工作台选择 Agent。
2. 设置目标平台、目标账号、知识库、语气和生成数量。
3. 执行标题生成、正文生成、平台改写、标签生成或审核辅助。
4. 在生成结果中采纳内容，写入内容项目。
5. 在 Agent 任务中心跟踪任务状态和成本。

产出：

- 多个候选生成结果。
- 写入内容项目的内容版本。
- Agent 执行记录。

### 阶段五：内容审核

1. 内容版本提交到审核中心。
2. 审核人员查看内容、平台、账号、风险等级和 AI 审核建议。
3. 审核通过则进入发布准备。
4. 审核驳回则回到内容项目修改，必要时重新调用 Agent。

产出：

- 审核任务。
- 审核结论。
- 可发布内容版本。

### 阶段六：发布准备与执行

1. 发布管理读取审核通过的内容版本。
2. 运营人员确认目标账号、发布时间和发布方式。
3. 平台生成对应发布准备：公众号草稿、小红书发布包、抖音发布包等。
4. 执行立即发布、定时发布或取消发布。
5. 发布完成后生成发布记录。

产出：

- 发布任务。
- 发布记录。
- 平台侧内容链接或状态。

### 阶段七：数据复盘

1. 发布后同步平台数据。
2. 按内容项目、平台、账号查看表现。
3. 生成数据复盘报告。
4. 将高表现标题、结构、素材、标签沉淀为知识库或 Prompt 优化建议。
5. 将表现差的内容反馈给选题和账号定位。

产出：

- 内容表现数据。
- 复盘报告。
- 知识库/Prompt/账号画像优化输入。

## 4. 数据流关系

```text
AccountProfile
  ├─ 影响 Agent 生成风格
  ├─ 约束发布目标账号
  └─ 支撑账号维度数据复盘

Topic
  └─ 创建 Content

Content
  ├─ 绑定 Material
  ├─ 生成 ContentVersion
  ├─ 创建 ReviewTask
  └─ 关联 AnalyticsReport

KnowledgeBase
  └─ 被 AgentRun 检索使用

Prompt
  └─ 被 Agent 执行时引用

AgentRun
  ├─ 输入 Content / Account / KnowledgeBase / Prompt
  └─ 输出候选结果或 ContentVersion

ReviewTask
  ├─ 通过后创建 PublishingTask
  └─ 驳回后回到 Content 修改

PublishingTask
  ├─ 执行发布生成 PublishRecord
  └─ 被 Analytics 同步消费

AnalyticsReport
  └─ 反哺 Topic / KnowledgeBase / Prompt / AccountProfile
```

## 5. 角色分工

### 管理员

- 配置系统参数、知识库、Prompt 和权限。
- 维护平台账号授权。
- 查看全局数据和异常。

### 运营人员

- 创建选题和内容项目。
- 调用 Agent 生成内容。
- 管理素材和发布计划。
- 查看发布结果和复盘报告。

### 审核人员

- 审核内容版本。
- 处理高风险内容。
- 给出驳回原因和修改建议。

### 数据/负责人

- 查看数据复盘。
- 总结高表现内容模式。
- 优化选题、账号定位、知识库和 Prompt。

## 6. 模块协作注意点

1. 平台账号是发布链路前置条件，未授权或授权失败的账号不能进入发布执行。
2. 内容项目是业务主实体，Agent、审核、发布、复盘都应围绕内容项目和内容版本流转。
3. Agent 生成结果不要直接发布，必须先写入内容版本并进入审核。
4. 审核通过是发布管理的准入条件。
5. 发布记录是数据复盘的准入条件。
6. 数据复盘结果应回流到知识库、Prompt 和账号画像，否则闭环不完整。
7. 知识库、素材库和账号画像越完整，Agent 生成质量越稳定。
